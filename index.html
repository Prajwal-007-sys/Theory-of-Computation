Formal Definition — DTM

M = (Q, Σ, Γ, δ, q₀, B, F)

Q
Finite set of states
Σ
Input alphabet ⊂ Γ−{B}
Γ
Tape alphabet, strict superset of Σ
δ
Q × Γ → Q × Γ × {L, R} (transition function)
q₀
Initial state (q₀ ∈ Q)
B
Blank symbol ∈ Γ − Σ
F
Set of accepting/final states (F ⊆ Q)
δ is a partial function — if undefined for (state, symbol), the TM halts (rejects/crashes).

How a DTM Operates

In one move, given current state q and scanned symbol X:

  1. Change state → move to new state q'
  2. Write symbol → overwrite X with Y on tape
  3. Move head → one cell Left or Right

δ(q, X) = (q', Y, D) — where D ∈ {L, R}

Acceptance & Rejection

✓ Acceptance

  • TM enters a final state → accepts immediately
  • No need to exhaust entire input!
  • Recognition is immediate when a final state is reached

✗ Rejection (Two ways)

  • Gets stuck — no δ defined for (state, symbol)
  • Loops infinitely — never reaches a final state

TM as Recogniser, Transducer, Enumerator

Recogniser

Given input w, TM accepts if w ∈ L, rejects or loops if w ∉ L.

Cannot always distinguish reject from loop!

Transducer (I/O Mapper)

TM can compute functions: given input x, produce output f(x) on tape when halting.

Used to model computable functions.

Enumerator

Generates all strings in L one by one using a separator # symbol.

L is RE ↔ L has an enumerator.

⚠ Common Mistake: A TM for RE language may loop forever on some inputs — it is NOT the same as a decider. A Decider (Recursive language) always halts — either accepting or rejecting.

Church-Turing Thesis

"There is an effective procedure for solving a problem if and only if there is a Turing Machine that solves it."

TMs formalise the intuitive notion of "algorithm". Any effectively computable function can be computed by a TM. This is a thesis, not a theorem — it cannot be formally proved.

Variants of Turing Machines

Multi-track, Semi-infinite tape, Multi-tape, Non-deterministic TM (NTM)

Key Principle: All variants are computationally equivalent to the standard DTM (Church-Turing). They may differ in efficiency but not in power.

1. Multi-track TM

Multiple tracks on a single tape. Each cell holds a tuple of symbols (one per track). The head reads/writes all tracks simultaneously.

Structure:

b
c

Two tracks; head reads column (c,0) simultaneously

Equivalence:

Simulate k tracks with 1 track by encoding each column as a tuple: (a,0), (b,1), (c,0)…

Tape alphabet Γ becomes Γ₁ × Γ₂ × … × Γₖ

2. Semi-infinite Tape

Tape is infinite only to the right (has a left end). Standard TM has two-way infinite tape.

Simulation with 2 tracks:

Track 1: original tape cells 0, 1, 2, 3, …

Track 2: original tape cells −1, −2, −3, … (reversed)

Special marker at position 0 separates the two halves.

Two-way infinite:
… −2 | −1 | 0 | 1 | 2 …

Semi-infinite (2 tracks):
Track 1: [*] [ 0] [ 1] [ 2] …
Track 2: [*] [−1] [−2] [−3] …

3. Multi-tape TM

k separate tapes each with own head. In one step, reads all k heads, writes to all tapes, moves each head independently.

Power: Same language class as DTM, but can be exponentially more efficient (time-wise).

Simulation by 1-tape DTM:

  1. Encode k tapes on one tape separated by #
  2. Mark current head position on each tape with special symbol
  3. Multiple passes to simulate one multi-tape step

Overhead: polynomial simulation — O(t²) single-tape steps for t multi-tape steps.

4. Non-deterministic TM (NTM)

At each step, δ may return a set of possible (state, symbol, direction) triples — the machine branches.

Acceptance:

NTM accepts if at least one computational branch reaches an accepting state.

Rejection:

NTM rejects only if every branch either:
  • Explicitly rejects (stuck in non-final state), OR
  • Goes into an infinite loop

Computation Tree:

      q₀
    /    \
  q₁   q₂
 / \   / \
q₃ q₄ q₅ q₆*

q₆* = accept state → NTM accepts!
NTM doesn't "choose" — it explores ALL branches in parallel (conceptually).

Equivalence with DTM

Every NTM can be simulated by a DTM using BFS on the computation tree. This avoids the problem of infinite loops in DFS.

⚠ Critical: Naive DFS simulation of NTM fails — if one branch loops infinitely, DFS will never explore other branches that may accept. Always use BFS!

BFS Simulation Algorithm:

k = 1
loop:
  for each string w₁, w₂, … in language:
    run TM for k steps on wᵢ
    if accepts → output "accept" and stop
  k++

Hilbert's Tenth Problem (Historical Context)

Problem: Given a Diophantine equation (polynomial with integer coefficients), does it have integer solutions?

1971 — Yuri Matiyasevich proved: No algorithm exists — this is undecidable.

This motivated the theory of undecidability we study next.

Decidability

Decidable problems, encoding TMs, ATM and the Halting Problem, undecidability proofs

Program = Data (Encoding)

A TM (or any program) can be represented as a string over some alphabet — its encoding.

We denote encoding of TM M as ⟨M⟩

We can give ⟨program, data⟩ to a general-purpose TM which runs program over data — this is the Universal Turing Machine (UTM).

Key insight: Programs can be treated as data. A TM can simulate another TM.

Encoding ⟨M, w⟩:

TM M encoded as binary string.

Input w encoded as binary string.

Together: ⟨M, w⟩ = encoding of "run M on w".

This is standard way to state decision problems.

A_TM — The Acceptance Problem

Theorem: A_TM = {⟨M,w⟩ | M is a TM and M accepts w} is undecidable.

But A_TM is RE: The Universal TM (UTM) recognises it — simulate M on w, if M accepts, UTM accepts.

Proof by Contradiction (Diagonalisation)

  1. Assume H is a decider for A_TM: H(⟨M,w⟩) = accept if M accepts w, reject otherwise
  2. Build D: on input ⟨M⟩, run H(⟨M,⟨M⟩⟩). If H accepts → D rejects. If H rejects → D accepts.
  3. Ask: what does D do on input ⟨D⟩?
    • If D accepts ⟨D⟩ → H(⟨D,⟨D⟩⟩) = accept → D rejects ⟨D⟩. Contradiction!
    • If D rejects ⟨D⟩ → H(⟨D,⟨D⟩⟩) = reject → D accepts ⟨D⟩. Contradiction!
  4. ∴ H cannot exist. A_TM is undecidable. ∎

The Halting Problem

HALT_TM = {⟨M,w⟩ | M is a TM and M halts on input w}

Proof HALT_TM is undecidable (by reduction from A_TM):

  1. Assume R decides HALT_TM
  2. Build decider S for A_TM: run R on ⟨M,w⟩; if R rejects (M doesn't halt) → reject; else simulate M on w; if M accepts → accept; else → reject
  3. S is a decider for A_TM — but A_TM is undecidable. Contradiction!
  4. ∴ HALT_TM is undecidable. ∎

Reducibility

Mapping reductions, showing undecidability, Rice's Theorem

Idea of Reduction

A reduces to B (written A ≤ₘ B) means: if we can solve B, we can solve A.

If A ≤ₘ B then:

  • B decidable → A decidable
  • A undecidable → B undecidable

Reduction diagram:

Input x → [f] → f(x) → [B] → answer

Computable function f maps instances of A to instances of B

x ∈ A ↔ f(x) ∈ B

Mapping Reduction (≤ₘ)

A ≤ₘ B via computable function f if: for all w, w ∈ A ↔ f(w) ∈ B

Properties:

  • If A ≤ₘ B and B is decidable → A is decidable
  • If A ≤ₘ B and A is undecidable → B is undecidable
  • If A ≤ₘ B and B is RE → A is RE
  • If A ≤ₘ B and A is not RE → B is not RE

Computable Function:

f: Σ* → Σ* is computable if there is a TM F that on input w, halts with f(w) on tape.

Classic Reduction Examples

Example 1: A_TM ≤ₘ HALT_TM

Show HALT_TM is undecidable by reducing A_TM to it.

Reduction f: f(⟨M,w⟩) = ⟨M',w⟩ where M' = M modified so that if M rejects, M' loops instead.

Then: M accepts w ↔ M' halts on w ↔ ⟨M',w⟩ ∈ HALT_TM.

Example 2: A_TM ≤ₘ E_TM (complemented)

Show E_TM = {⟨M⟩ | L(M) = ∅} is undecidable.

Reduction: Given ⟨M,w⟩, construct M_x that accepts only if x=w AND M accepts w.

L(M_x) ≠ ∅ ↔ M accepts w. So ⟨M,w⟩ ∈ A_TM ↔ ⟨M_x⟩ ∉ E_TM.

Example 3: A_TM ≤ₘ REGULAR_TM

REGULAR_TM = {⟨M⟩ | L(M) is a regular language} — undecidable.

Idea: Build M₂ that if w is of form 0ⁿ1ⁿ accepts, else runs M on its original input. If M accepts w, L(M₂) = Σ* (regular). If M rejects, L(M₂) = {0ⁿ1ⁿ} (not regular). A decider for REGULAR_TM would decide A_TM. Contradiction.

Rice's Theorem

Statement: Let P be any non-trivial property of the language of a TM (i.e. depends only on what strings are accepted). Then the language {⟨M⟩ | L(M) has property P} is undecidable.

What is non-trivial?

  • P is non-trivial if there exist TMs M₁, M₂ such that L(M₁) ∈ P and L(M₂) ∉ P

Examples of properties covered by Rice's:

  • {⟨M⟩ | L(M) is regular} — undecidable
  • {⟨M⟩ | L(M) is CFL} — undecidable
  • {⟨M⟩ | L(M) is finite} — undecidable
  • {⟨M⟩ | ε ∈ L(M)} — undecidable
Rice's ≠ all properties undecidable.
Only semantic (language) properties. Syntactic properties like "M has 5 states" are decidable!
Using Rice's in an exam:
1. Identify the property P of L(M)
2. Check P is non-trivial
3. Conclude: undecidable by Rice's theorem

Summary: Reduction Strategy

Goal Method
Show B is undecidable Find known undecidable A, show A ≤ₘ B
Show B is not RE Find A not RE, show A ≤ₘ B
Show B is decidable Give an explicit TM that decides B
Show B is RE (not decidable) Give recogniser, then show complement not RE

Critical Theorems

Theorem 1

If L and L̄ (complement) are both RE, then L is Recursive (R).

Proof: Run TM₁ (for L) and TM₂ (for L̄) in parallel. One must halt. If TM₁ accepts → accept; if TM₂ accepts → reject. Always halts.

Theorem 2 (Contrapositive)

If L ∈ RE but L ∉ R, then L̄ ∉ RE.

Example: A_TM ∈ RE, A_TM ∉ R → complement of A_TM ∉ RE.
