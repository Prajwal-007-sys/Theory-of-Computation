HALTING PROBLEM — LIVE EXPLANATION
Core Theorem
There is no algorithm that, given any program P and input x, decides whether P(x) terminates.
This is not a limitation of current technology — it is a fundamental mathematical impossibility, proved by Turing in 1936.
Proof Sketch
1. Assume oracle H(P,x) exists that returns HALT or LOOP.
2. Build machine D(P): calls H(P,P), then does the opposite.
3. Ask: what does D(D) do?
If D(D) halts → H said LOOP → D loops → contradiction. If D(D) loops → H said HALT → D halts → contradiction. ∴ H cannot exist.

No algorithm decides whether an arbitrary Diophantine equation (polynomial, integer coefficients) has integer solutions. Every recursively enumerable set is Diophantine (DPRM), so a decision procedure would solve the Halting Problem.

Why No Algorithm Exists

DPRM encodes TM computations as Diophantine constraints. A decision procedure would resolve Halting — impossible. The search space is infinite; solutions may exist at arbitrarily large integers, with no computable bound separating solvable from unsolvable.

HILBERT'S 10TH — LIVE EXPLANATION
What is a Diophantine Equation?
A polynomial equation with integer coefficients where we seek integer solutions. Named after ancient Greek mathematician Diophantus.
Current: x² + y² = z² (Pythagorean)
DPRM Theorem
Davis-Putnam-Robinson (1950–61) + Matiyasevich (1970) proved:
Every recursively enumerable set S is Diophantine: x ∈ S ↔ ∃ integers satisfying some polynomial p(x, y₁,…,yₖ) = 0
Since the Halting Set is r.e. but undecidable, Diophantine solvability is also undecidable.

Reduction from Halting
Post showed every TM computation can be encoded as a PCP instance. A solution corresponds to a valid computation history. Since Halting is undecidable, PCP is too.
Key insight: tiles simulate TM tape symbols, states, and transition steps — a match = acceptance.
Why BFS Is Insufficient
BFS can find solutions when they exist (semi-decide). But it cannot confirm no solution exists — the search may run forever. This is exactly why PCP is only semi-decidable.
Variants & Relations
• Binary PCP (2-letter alphabet): still undecidable
• Bounded PCP (max sequence length k): decidable
• Generalised PCP: used in undecidability proofs for CFG intersection, ambiguity, and more

Berger's Construction
Each TM configuration maps to a tile colour pattern. A valid tiling of the infinite plane encodes an infinite (non-halting) computation. If the TM halts, no infinite tiling exists.
Halting TM ↔ Tile set cannot tile ℤ². Since halting is undecidable, so is tiling.
Aperiodic Tilings
Some tile sets can only tile the plane aperiodically — without any translational symmetry. Wang originally conjectured no such sets exist. Berger's proof of undecidability showed they must.
Famous examples: Penrose tiles (2 tiles), Wang tiles (hundreds, then reduced to 11 by Jeandel-Rao 2021).

P vs NP — LIVE EXPLANATION
The Central Question
Can every problem whose solution can be verified quickly also be solved quickly?
P ⊆ NP is known. Whether P = NP or P ≠ NP is the biggest open question in computer science.

Comparison-based sorting: O(n log n) is optimal. Merge sort and heapsort achieve this.
TM Complexity Model
Deterministic TM: at each step, exactly one transition fires. Time = steps to halt.
Non-deterministic TM: can branch into all possible transitions simultaneously. Accepts if any branch accepts.
NDTM can "guess" a solution in O(1) and verify in poly-time. This is exactly NP.
Consequences if P = NP
• RSA and elliptic-curve cryptography would break
• Protein folding, drug discovery: instant optimal solutions
• Mathematical proof discovery automated
• Travelling salesman, scheduling: exact solutions in seconds
Cook-Levin Theorem (1971)
SAT is NP-complete — every NP problem reduces to SAT in polynomial time. If SAT ∈ P, then P = NP. SAT remains the canonical NP-complete problem.
